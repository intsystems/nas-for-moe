## Что должен делать итоговый код

### 1. Входные данные

Код должен принимать:

* обучающий датасет;
* вектор кластеризации длины `N_train`, где для каждой обучающей точки указан номер кластера;
* число кластеров `K`;
* пространство архитектур / механизм перебора или сэмплирования архитектур.

### 2. Логика подмножеств кластеров

Каждая конфигурация обучения определяется:

* архитектурой;
* бинарным вектором `b ∈ {0,1}^K`.

Смысл `b`:

* `b[k] = 1` означает, что кластер `k` включён в обучающее подмножество;
* `b[k] = 0` означает, что кластер `k` исключён.

То есть модель обучается только на тех обучающих примерах, чьи кластеры включены в `b`.

### 3. Что предсказывает суррогатная функция

Суррогатная функция должна принимать на вход:

* архитектуру;
* бинарный вектор `b`.

И выдавать:

* **валидационную точность** для этой архитектуры, если обучать её только на примерах из кластеров, выбранных в `b`.

### 4. Как считать валидацию

Для валидационных точек кластер определяется так:

* каждая валидационная точка относится к тому кластеру, **центр которого ближе всего** по расстоянию.

Далее качество модели можно считать:

* с использованием этой кластерной разметки для согласованного подсчёта по тем же кластерам.

### 5. Активный сбор новых точек для датасета

Алгоритм выбора следующей точки для вычисления должен быть таким:

1. Сначала выбираются **случайные подмножества кластеров** `b`.
2. Для каждого такого `b` рассматриваются кандидаты-архитектуры.
3. Следующая архитектура выбирается по критерию:
   [
   \mu + \sigma
   ]
   где:

   * `mu` — среднее предсказание суррогатной функции,
   * `sigma` — неопределённость предсказания.

### 6. Как считать неопределённость

Для оценки `sigma` используй **MC Dropout**:

* включай dropout на инференсе;
* делай несколько стохастических прогонов;
* по набору предсказаний считай среднее `mu` и стандартное отклонение `sigma`.

### 7. Итеративный цикл

Нужен следующий цикл:

1. Сэмплируются начальные пары:

   * подмножество кластеров `b`,
   * архитектуры для каждого вектора.
2. Для них реально обучаются модели и измеряется валидационная точность.
3. Из этих точек собирается датасет для суррогатной функции.
4. Суррогатная функция обучается.
5. Затем снова сэмплируются новые кандидаты:

   * сначала случайные `b`,
   * затем архитектуры выбираются через максимум `mu + sigma`.
6. Для выбранных кандидатов снова реально считается качество.
7. Новые точки добавляются в датасет.
8. Цикл повторяется.
