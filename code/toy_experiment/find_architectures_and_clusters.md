## Контекст

* сборки датасета архитектур,

Суррогатная функция `u(alpha, R)` предсказывает, насколько хорошо архитектура `alpha` подходит для работы на данных при заданном векторе принадлежности кластеров `R`.

Важно:
суррогатная функция обучалась **только на дискретных бинарных векторах**.
Поэтому:

* если на вход суррогатной функции подаётся вероятностный / непрерывный вектор, нужно **сэмплировать дискретный вектор** на его основе;
* если в алгоритме нужна дифференцируемость по такому вектору, нужно использовать **Gumbel-Softmax / straight-through relaxation** или близкий практический аналог.

## Математическая постановка

Я хочу максимизировать следующее правдоподобие:

[
\prod_{m=1}^{M} \sum_{k=1}^{K} r_{mk}; u(\alpha_k, R_k)
\to
\max_{\alpha_1,\dots,\alpha_K,; R_1,\dots,R_K}
]

Где:

* `M` — количество кластеров данных;
* `K` — количество экспертов;
* `u(alpha_k, R_k)` — значение суррогатной функции для архитектуры `alpha_k` и вектора `R_k`;
* `r_{mk}` — вероятность того, что кластер данных `m` назначается эксперту `k`;
* `R_k` — вектор вероятностей / весов назначения кластеров данных эксперту `k`.

### Важная семантика переменных

Нужно аккуратно различать:

* `m` — индекс кластера данных;
* `k` — индекс эксперта;
* `r_{mk}` — вероятность назначения кластера `m` эксперту `k`;
* строка `r[m, :]` должна задавать распределение по экспертам для фиксированного кластера `m`;
* итоговое дискретное назначение кластеров экспертам получается как:
  [
  \operatorname{argmax}*k r*{mk}
  ]
* `R_k` — вектор длины `M`, описывающий, насколько каждый кластер данных относится к эксперту `k`, то есть по сути столбец матрицы `r[:, k]`, возможно в непрерывной или релаксированной форме.

То есть:

* матрица `r` имеет размер `[M, K]`;
* `R_k = r[:, k]`.

# Метод 1. Полный / частичный поиск через сэмплирование

Идея:

* сэмплировать кандидаты матрицы `r` / дискретных назначений кластеров экспертам;
* сэмплировать архитектуры экспертов `alpha_1, ..., alpha_K`;
* вычислять значение целевой функции;
* выбирать лучший вариант.

### Требования

* Поддержать разумное число сэмплов;
* Если полное пространство слишком велико, использовать Monte Carlo / random search;
* Для каждого эксперта использовать соответствующий `R_k`;
* Поскольку суррогат обучен на дискретных векторах, при обращении к нему использовать корректный бинарный вектор.

---

# Метод 2. Градиентная оптимизация по `r` + сэмплирование архитектур

Идея:

* оптимизировать целевую функцию напрямую;
* по `r_{mk}` / `R_k` использовать автоматическое дифференцирование;
* архитектуры `alpha_k` всё равно подбирать сэмплированием на каждой итерации или через отдельный search step.

### Требования

* Параметризовать `r` так, чтобы по каждой строке получалось корректное распределение вероятностей (например, через logits + softmax);
* Для передачи `R_k` в суррогат:

  * либо использовать дискретное сэмплирование;
  * либо Gumbel-Softmax / straight-through, если нужна дифференцируемость;
* Явно прокомментировать, как именно реализована дифференцируемая аппроксимация;
* На каждой итерации или каждые несколько итераций выполнять search по архитектурам `alpha_k`.

---

# Метод 3. EM-алгоритм

Использовать следующую схему:

### E-step

[
q_{mk} \propto r_{mk} \cdot u(\alpha_k, R_k)
]

с нормализацией по `k` для каждого `m`.

### M-step

Максимизировать:
[
\sum_{m=1}^{M} \sum_{k=1}^{K} q_{mk}\log\left(r_{mk}\cdot u(\alpha_k, R_k)\right)
]

по:

* архитектурам `alpha_k`,
* матрице `r` / векторам `R_k`.

### Требования

* На M-шаге:

  * по `alpha_k` использовать сэмплирование / search;
  * по `R_k` и/или `r` использовать дифференцирование;
* Если потребуется релаксация, использовать Gumbel-Softmax;
* Нужно явно описать в коде, где E-step, где M-step.
