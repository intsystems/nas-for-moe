# Датасет `data/`

Путь: `code/toy_experiment/data/`

Создан скриптом `toy_generate_dataset.py` (с параметрами по умолчанию, seed=322), затем кластеризован `toy_clustering.py` (20 кластеров, seed=322).

## Генерация (toy_generate_dataset.py)

Синтетические 2D-данные для классификации на **4 класса**: **1 linear cloud (2 класса) + 1 ring cloud (2 класса)**, balanced.

- Классы 0, 1 — linear cloud (разделены диагональю x > y)
- Классы 2, 3 — ring cloud (inner / outer кольцо)

| Параметр | Значение |
|---|---|
| n_samples (на тип) | 1000 |
| seed | 322 |
| linear_scale | 2.0 |
| linear_offset | -6.0 |
| ring_center_x | 2.0 |
| inner_radius | 1.0 |
| outer_radius | 2.0 |
| ring_noise | 0.3 |

## Кластеризация (toy_clustering.py)

KMeans с train/val split.

| Параметр | Значение |
|---|---|
| n_clusters | 20 |
| test_size | 0.2 |
| seed | 322 |

## Идеальное разбиение (ideal split)

Идеальное разбиение (`ideal_split.png`, построено скриптом `plot_ideal_split.py`) — это разделение кластеров на два эксперта по типу облака данных:

- **Expert 0 (linear):** кластеры, центроид которых имеет `x0 < -2.0` — соответствуют данным из линейного облака (классы 0, 1).
- **Expert 1 (ring):** кластеры, центроид которых имеет `x0 >= -2.0` — соответствуют данным из кольцевидного облака (классы 2, 3).

Порог `x0 = -2.0` выбран вручную как граница между двумя типами облаков (может потребовать корректировки после перекластеризации). Это разбиение является ожидаемым оптимальным результатом оптимизации MoE с `K=2` экспертами: один эксперт специализируется на линейных данных, другой — на кольцевидных.
