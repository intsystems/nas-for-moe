# Теория: оптимизация назначений экспертов в MoE

## Математическая постановка

Максимизировать правдоподобие MoE:

```
∏_{m=1}^{M} Σ_{k=1}^{K} r_{mk} · u(α_k, R_k)  →  max_{α_1,...,α_K, R_1,...,R_K}
```

Где:
- `r` — матрица `[M, K]`, `r[m, k]` — вероятность назначения кластера `m` эксперту `k`
- `R_k = r[:, k]` — столбец матрицы, вектор принадлежности кластеров эксперту `k`
- `u(α_k, R_k)` — суррогатная функция (см. @docs/surrogate.md)

## Три метода оптимизации

### Метод 1: Sampling search

- Сэмплировать кандидаты матрицы `r` / дискретных назначений
- Сэмплировать архитектуры `alpha_1, ..., alpha_K`
- Вычислить objective, выбрать лучший вариант
- При большом пространстве — Monte Carlo / random search
- Суррогату подавать дискретные бинарные вектора

### Метод 2: Градиентная оптимизация по `r` + сэмплирование архитектур

- Параметризовать `r` через logits + softmax
- Для передачи `R_k` в суррогат: Gumbel-Softmax / straight-through
- Архитектуры `alpha_k` подбирать сэмплированием
- На каждой итерации / каждые N итераций — search по архитектурам

### Метод 3: EM-алгоритм

**E-step:**
```
q_{mk} ∝ r_{mk} · u(α_k, R_k)
```
с нормализацией по `k` для каждого `m`.

**M-step:**
Максимизировать:
```
Σ_m Σ_k q_{mk} · log(r_{mk} · u(α_k, R_k))
```
по `alpha_k` (сэмплированием) и по `r` / `R_k` (дифференцированием).

## Единый интерфейс методов

- **Вход:** `K`, суррогат, данные, кластеризация, sampler архитектур, гиперпараметры
- **Выход:** лучшие `alpha_1..K`, матрица `r [M,K]`, дискретное назначение (argmax), значение objective, история оптимизации
