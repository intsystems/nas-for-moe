\section{Problem Statement}
\label{sec:problem_statement}

\subsection{Search Space of MoE}
\label{subsec:searchspace}

Each expert is a convolutional network assembled from a \emph{cell}: a
directed acyclic graph whose nodes are computation steps. We use the
cell encoding of our implementation, in which a cell has a fixed number
of nodes and each node applies a single operation, drawn from a fixed
finite set $\mathcal{O}$ (convolutions of different kernel sizes,
pooling, and a skip connection), to the output of one preceding node;
the first node reads the cell input. An architecture is therefore
determined by two discrete choices per node --- \emph{which} operation
the node applies and \emph{into which} node it sends that operation,
i.e.\ which earlier node feeds into it --- and we gather all these
choices into a single architecture vector $\boldsymbol{\alpha}$, so that
$\boldsymbol{\alpha}$ describes one concrete cell. Since both
$\mathcal{O}$ and the number of nodes are finite, the search space
$\mathcal{A}$ of all admissible $\boldsymbol{\alpha}$ is finite, and
neural architecture search reduces to selecting the best
$\boldsymbol{\alpha} \in \mathcal{A}$. For an architecture
$\boldsymbol{\alpha}_k$ with trained weights $\boldsymbol{\theta}_k$ we
denote the resulting network by
$f(\boldsymbol{\alpha}_k, \boldsymbol{\theta}_k): \mathcal{X} \to \mathcal{Y}$.

A Mixture-of-Experts (MoE) model combines $K$ such experts, each meant
to specialise on a different region of the input space. Stacking the
expert architectures into
$\boldsymbol{\alpha} = [\boldsymbol{\alpha}_1^{\!\top}, \ldots, \boldsymbol{\alpha}_K^{\!\top}]^{\!\top}$,
the MoE prediction is the routing-weighted combination of expert outputs
\begin{equation}
\label{eq:moe_pred}
g(\boldsymbol{\alpha}, \boldsymbol{\theta}, \boldsymbol{w}_r)(\boldsymbol{x})
= \sum_{k=1}^{K} r_k(\boldsymbol{x}, \boldsymbol{w}_r)\, f(\boldsymbol{\alpha}_k, \boldsymbol{\theta}_k)(\boldsymbol{x}),
\end{equation}
where the gate values $r_k(\boldsymbol{x}, \boldsymbol{w}_r) \geq 0$ with
$\sum_{k=1}^{K} r_k(\boldsymbol{x}, \boldsymbol{w}_r) = 1$ are produced by
a routing network with parameters $\boldsymbol{w}_r$.

\textbf{Data clustering and routing.}
Prior to architecture search, the dataset is partitioned into $M$ clusters,
$\mathcal{D} = \mathcal{C}_1 \cup \mathcal{C}_2 \cup \cdots \cup \mathcal{C}_M$,
each grouping samples of similar structure (e.g.\ a semantic class group or a data modality). Throughout the paper, $n \in \{1, \ldots, N\}$ indexes individual dataset samples, $m \in \{1, \ldots, M\}$ indexes clusters, and $k \in \{1, \ldots, K\}$ indexes experts. We write $\mathcal{C}_m \subseteq \{1, \ldots, N\}$ for the sample indices that belong to cluster $m$, and denote by $m(n) \in \{1, \ldots, M\}$ the cluster index of sample $n$.

Routing is defined at the cluster level: let $r_{mk} \in [0, 1]$ be the probability that cluster $m$ is assigned to expert $k$, with $\sum_{k=1}^{K} r_{mk} = 1$, collected in a routing matrix $\mathbf{R} \in [0, 1]^{M \times K}$. The column $\mathbf{r}_k = \mathbf{R}_{:,k} \in [0, 1]^M$ encodes the clusters expert $k$ is responsible for; the hard cluster$\to$expert assignment recovered at the end of optimisation is $I_k = \{m : \arg\max_{k'} r_{mk'} = k\} \subseteq \{1, \ldots, M\}$, the set of cluster indices routed to expert $k$.

Unlike standard MoE training, where only the gating parameters $\boldsymbol{w}_r$ are learned for a fixed architecture, here \emph{finding the routing $\mathbf{R}$ is itself part of the architecture search}: the search jointly determines the per-expert architectures $\boldsymbol{\alpha}_{1:K}$ and the assignment of clusters to experts. This couples architecture selection with data specialization, since the best architecture for an expert depends on the cluster subset it is trained on.

\subsection{MoE Objective}
\label{subsec:objective}

We cast MoE architecture search as a likelihood-maximization problem. The goal is to choose, for every expert $k$, an architecture $\boldsymbol{\alpha}_k$, its trained parameters $\boldsymbol{\theta}_k$, and the cluster index set $I_k \subseteq \{1, \ldots, M\}$ it is responsible for, so as to maximize the data likelihood under the mixture:
\begin{equation}
\label{eq:moe_likelihood}
\prod_{n=1}^{N} \sum_{k=1}^{K} p\bigl(y_n,\, m(n) \in I_k \,\big|\, x_n, \boldsymbol{\alpha}_k, \boldsymbol{\theta}_k\bigr)
\;\to\; \max_{\boldsymbol{\alpha}_{1:K},\, \boldsymbol{\theta}_{1:K},\, I_{1:K}},
\end{equation}
where the event $m(n) \in I_k$ encodes ``sample $n$'s cluster is routed to expert $k$''.

Each joint term factorizes into a routing probability and a conditional likelihood:
\begin{equation}
\label{eq:moe_factor}
p\bigl(y_n,\, m(n) \in I_k \,\big|\, x_n, \boldsymbol{\alpha}_k, \boldsymbol{\theta}_k\bigr)
= p\bigl(m(n) \in I_k\bigr)\, p\bigl(y_n \,\big|\, x_n, \boldsymbol{\alpha}_k, \boldsymbol{\theta}_k,\, m(n) \in I_k\bigr).
\end{equation}
The membership probability is precisely the cluster-level routing,
$p\bigl(m(n) \in I_k\bigr) = r_{m(n), k}$. For the conditional factor we adopt the standard MLE proxy: since the experts are trained by minimising cross-entropy, the average per-sample negative log-likelihood (NLL) of expert $k$ on its assigned data is exactly what training drives down. We therefore introduce a \emph{surrogate function}
\begin{equation}
\label{eq:surrogate_def}
u: \mathcal{A} \times [0,1]^M \to \mathbb{R}_{\geq 0},
\qquad u(\boldsymbol{\alpha}_k, \mathbf{r}_k)
\;\approx\;
- \frac{1}{|\mathcal{D}_{\mathbf{r}_k}^{\text{val}}|}\!\!
\sum_{n \in \mathcal{D}_{\mathbf{r}_k}^{\text{val}}}
\!\!\log p\bigl(y_n \,\big|\, x_n, \boldsymbol{\alpha}_k, \boldsymbol{\theta}_k^{*}\bigr),
\end{equation}
that predicts the \emph{mean per-sample cross-entropy loss} of architecture $\boldsymbol{\alpha}_k$ trained on the cluster subset encoded by $\mathbf{r}_k$ (here $\boldsymbol{\theta}_k^{*}$ are the parameters obtained by training $\boldsymbol{\alpha}_k$ on $\mathcal{D}_{\mathbf{r}_k}$, and $\mathcal{D}_{\mathbf{r}_k}^{\text{val}}$ is the corresponding validation subset). A concrete construction of $u$ as a learned regressor is given in Section~\ref{subsec:surr}.

The surrogate is treated as constant within a cluster: for every $n$ in cluster $m$ assigned to expert $k$,
\begin{equation}
\label{eq:moe_persample_lik}
p\bigl(y_n \,\big|\, x_n, \boldsymbol{\alpha}_k, \boldsymbol{\theta}_k,\, m(n) \in I_k\bigr)
\;\approx\; e^{-u(\boldsymbol{\alpha}_k, \mathbf{r}_k)},
\end{equation}
i.e.\ $u$ is the per-sample geometric mean of the conditional likelihood. Substituting~\eqref{eq:moe_factor} and~\eqref{eq:moe_persample_lik} into~\eqref{eq:moe_likelihood} yields the surrogate-based objective
\begin{equation}
\label{eq:moe_surr_likelihood}
\prod_{n=1}^{N} \sum_{k=1}^{K} r_{m(n), k}\, e^{-u(\boldsymbol{\alpha}_k, \mathbf{r}_k)}
\;\to\; \max_{\boldsymbol{\alpha}_{1:K},\, \mathbf{r}_{1:K}}.
\end{equation}

Because routing is decided at the cluster level, $r_{m(n), k} = r_{m, k}$ for every $n \in \mathcal{C}_m$, and $u(\boldsymbol{\alpha}_k, \mathbf{r}_k)$ is identical across all samples of a cluster. Taking the logarithm of~\eqref{eq:moe_surr_likelihood} and grouping the $N$ per-sample terms by cluster collapses each cluster's $|\mathcal{C}_m|$ identical log-factors into a single weighted term, yielding the \emph{cluster-aware MoE objective}:
\begin{equation}
\label{eq:moe_objective}
\mathcal{J}(\boldsymbol{\alpha}_{1:K}, \mathbf{R}) \;=\;
\sum_{m=1}^{M} |\mathcal{C}_m|\,\log \sum_{k=1}^{K} r_{mk}\, e^{-u(\boldsymbol{\alpha}_k, \mathbf{r}_k)}
\;\to\; \max_{\boldsymbol{\alpha}_{1:K},\, \mathbf{R}}.
\end{equation}
Intuitively, $r_{mk}$ specifies how strongly cluster $m$ is routed to expert $k$, $e^{-u(\boldsymbol{\alpha}_k, \mathbf{r}_k)}$ is the per-sample likelihood that expert $k$ would attain on its assigned cluster subset, and the cluster-size weight $|\mathcal{C}_m|$ reflects that a cluster's contribution to the joint log-likelihood scales with the number of samples it contains. Problem~\eqref{eq:moe_objective} is solved by the algorithm of Section~\ref{subsec:sgem} via Generalised EM with an adaptively refined surrogate.
